平台层

提供操作系统/平台相关的底层功能

包括：平台检测、原子数据类型、集合及迭代器、文件系统、网络传输层(UDP/TCP)、高分辨率时钟、线程库、图形包裹类、物理/碰撞包裹类

文件系统

1、 引擎包装原生的文件系统，能够对系统其他部分产生隔离作用，隐藏不同目标平台之间的区别
2、 操作系统的文件系统API未必能提供游戏引擎所需的功能(许多引擎支持串流，能够在游戏运行中同时载入数据)，但多数操作系统不直接提供串流功能的文件系统API

提供的功能

1、操作文件名和路径
2、开、关、读、写单独的文件
3、扫描目录下的内容
4、处理异步文件输入/输出(I/O)请求(串流用)

1、文件名和路径

路径通常包含一个可选的卷指示符紧接着一串路径成分，之间以路径分隔符分割
在路径中要指明根目录，通常是有可选的卷指示符连接路径分隔符
(Unix中的路径分隔符 / )
(Windows中的路径分隔符 C:\ )

  • 绝对路径和相对路径
  • 搜寻路径

    1. 路径是代表文件系统下某文件或目录的字符串。搜寻路径是含一串路径的字符串,各路径之间以特殊字符 (如冒号或分号)分隔找文件时就会从这些路径进行搜寻。例如在命令行下执行程序,操作系统会首先看看当前目录下有没有该可执行文件若没有则会从 PATH 环境变量中的路径搜寻该可执行文件。

    2. 有些游戏引擎会使用搜寻路径找资源文件。例如，OGRE 染引擎有一个 resources.cfg 文本文件,它会使用其中的搜寻路径。此文件包含目录及 ZIP 存档列表要找资产时就会从该列表按序进行搜寻。然而,在运行时搜寻资产可能是费时的做法。通常,资产路径在运行之前就可得知。因此我们应能完全避免搜寻资产这显然是更优越的做法

  • 路径API
    从路径中分离目录/文件名/拓展名、使路径规范化、在绝对路径和相对路径之间进行转换等

2、基本文件I/O

  1. 标准C程序库(standard C library) 提供了两组 API 以开启读取及写文件内容两组API中一组有缓冲功能(buffered),另一组无缓冲功能(unbuffered)。每次调用输入/输出(in-put/output,I/0) 时,都需要称为缓冲区的数据区块,以供程序和磁盘之间传送来源或目的字节。
  2. 当API负责管理所需的输入/输出数据缓冲时,就称之为有缓冲功能的I/O API相反若需要由程序员负责管理数据缓冲时则称之为无缓冲功能的API。在 C标准程序库中有I/0缓冲功能的函数有时候会被称为流输入/输出(stream I/0)API因为这些API把磁盘文件抽象成字节流

3、异步文件I/O

为了支持串流，必须使用异步文件I/O库，这种库能让程序在请求I/O后，不需要等待读写完成，程序便立即继续运行。

  1. 优先权
    异步I/O系统必须能够暂停较低优先权的请求，才可以让较高优先权的I/O请求有机会在时限到达前完成
  2. 如何工作
    异步文件是利用利用另一个单独线程处理I/O请求的。主线程调用异步函数时，会把请求放入一个队列，并立即传回。同时,I/0 线程从队列中取出请求,并以阻塞 (blocking)I/0 函数如 read()或 fread()处理这些请求。请求的工作完成后就会调用主线之前提供的回调函数，告之该操作已完成。若主线选择等待完成 I/0 请求就会使用信号量(seaphore)处理。(每个请求对应一个信号量，主线程把自身处于休眠状态等待 I/0线程在完成请求工作后通知信号量。)
